`nfr-review arch` scans one or more repositories and produces architecture
documentation including technology maps, component diagrams, domain models,
and optionally a [Structurizr DSL](https://docs.structurizr.com/dsl) workspace.

## Usage

```bash
# Single repo â€” generates JSON + Markdown + PDF
nfr-review arch /path/to/repo

# Multiple repos as a unified report
nfr-review arch /path/to/repo1 /path/to/repo2

# Generate Structurizr DSL output
nfr-review arch --format dsl /path/to/repo

# Skip LLM-based analysis
nfr-review arch --no-llm /path/to/repo
```

## Options

| Flag | Default | Description |
|------|---------|-------------|
| `--output-dir PATH` | `reports` | Directory where report files are written |
| `--format FORMAT` | `json` + `md` + `pdf` | Output format(s): `json`, `md`, `pdf`, `dsl`. Repeat for multiple |
| `--no-llm` | off | Skip LLM-based analysis (domain model, market comparison) |
| `--diagram-mode MODE` | `hierarchical` | Component diagram layout: `hierarchical` or `flat` |
| `--evidence-dir DIR` | â€” | Directory containing evidence JSONL files (e.g. OTel traces) |
| `-v` / `-q` / `--log-file` | â€” | Verbosity controls, same as the `run` command |

## What it produces

The `arch` command discovers components, integrations, and infrastructure from
source code, manifests, and configuration files. It then generates:

- **Technology map** â€” detected languages, frameworks, infrastructure, and CI/CD
  tools with file counts and locations
- **Domain model** â€” extracted entity relationships and bounded contexts
  (LLM-enhanced when a backend is configured)
- **C4 diagrams** â€” Mermaid architecture diagrams at context, container,
  component, and code levels
- **Risk assessment** â€” architectural risks and recommendations
- **Multi-repo view** â€” when given multiple repos, produces a cross-repo
  architecture overview

## Output formats

### JSON

Structured `ArchReport` object with all components, integrations, diagrams,
and metadata. Suitable for programmatic consumption.

### Markdown

Human-readable report with embedded Mermaid diagrams, technology tables,
risk assessments, and recommendations.

### PDF

Rendered version of the Markdown report via WeasyPrint. Requires the `pdf`
extra: `pip install "nfr-review[pdf]"`.

### Structurizr DSL

A [C4 model](https://c4model.com/) workspace file that can be:

- Rendered with [Structurizr Lite](https://docs.structurizr.com/lite)
- Exported to PNG/SVG via the [Structurizr CLI](https://docs.structurizr.com/cli)
- Version-controlled in Git to track architectural evolution
- Diffed between scans to detect architectural drift

```bash
nfr-review arch --format dsl /path/to/repo
# Output: reports/arch-workspace.dsl
```

## Component discovery

The arch command discovers components from multiple sources:

| Source | What it finds |
|--------|--------------|
| Build files (`pom.xml`, `build.gradle`, `pyproject.toml`, `go.mod`) | Project modules and their technologies |
| Docker Compose | Service definitions and their relationships |
| Kubernetes manifests | Deployments, services, and config maps |
| Helm charts | Chart metadata and value schemas |
| OTel trace evidence | Runtime service topology (when `--evidence-dir` is provided) |

Integrations (database connections, API calls, message queues) are inferred
from configuration files, connection strings, and dependency declarations.

## Diagram modes

The `--diagram-mode` flag controls component diagram layout:

- **`hierarchical`** (default) â€” generates an overview diagram plus detailed
  diagrams for each major component. Better for large systems.
- **`flat`** â€” single diagram showing all components. Better for smaller systems
  or quick overviews.

## LLM dependency

Without an LLM backend, `arch` produces technology maps, component analysis,
and static C4 diagrams. With an LLM configured, it additionally generates:

- Enhanced domain model with bounded context analysis
- Market comparison showing how the architecture compares to common patterns

See [Advanced Features](/learn/10-advanced#llm-integration) for LLM setup.
