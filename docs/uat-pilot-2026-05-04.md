# UAT: Pilot Validation against agentic-java-demo

**Date:** 2026-05-04
**Target repo:** `../agentic-java-demo` (Java/Spring Boot microservice with K8s, CI, security)
**Tool version:** 0.1.0
**Rules registered:** 20 (18 Band 1 deterministic + 2 Band 2 LLM-augmented)

## Target Repo Profile

- **Java:** Rich source tree — controllers, services, config, security, resilience patterns
- **K8s:** Full Kustomize setup with base + overlays (prod/test/dev/azure), includes NetworkPolicy
- **Spring Boot:** 10 config files across dev/prod/test/docker/kubernetes profiles
- **CI:** 18 GitHub Actions workflows
- **ADR/Docs:** No formal ADR directory (extensive architecture docs, but not in ADR format)
- **APIM:** Not present

Ran with defaults — no `nfr-review.yaml`, no tech filtering.

- **Result:** 20 rules loaded, 12 ran, 8 skipped, 48 findings
- **Observation:** Spring-specific rules skipped because `tech_present` was not declared. APIM rules correctly skipped via tech filtering.

### Phase 3: Config Tuning

Created `nfr-review.yaml` declaring `spring_boot` and `kubernetes` in `tech_present`, skipping `health-endpoint-missing` (known false positive due to Actuator).

- **Result:** 14 rules ran, 6 skipped, 50 non-skipped findings, 0.49 seconds
- **Spring rules now active:**
  - `actuator-exposure-risk`: amber/medium — dev profile exposes `*` endpoints (real finding)
  - `logging-config-missing`: amber/low — no structured logging detected (debatable)
  - `spring-profile-misconfiguration`: green — prod profile clean

### Phase 4: Performance

- **Scan time:** 0.49 seconds (target: under 2 minutes)
- Performance is excellent with room to spare.

## Known Improvements (Backlog)

1. **`non-root-container-violation`** — Does not check pod-level `securityContext` inheritance. Pod-level `runAsNonRoot: true` should satisfy the check for all containers. (4 false positives in this pilot)
2. **Resilience rule on test classes** — `resilience-annotation-missing` fires on `src/test/` files. Should exclude test paths or filter by `src/main/` only.
3. **`health-endpoint-missing` + Actuator** — Rule only checks for explicit `@RestController` methods mapping `/health`. Does not account for Spring Boot Actuator auto-registration of `/actuator/health`.
4. **PII summary text** — LLM-confirmed PII findings could include more descriptive summaries about what the LLM found and why it flagged the code.

## Verdict

**Pass.** The tool produces real, actionable findings against a production-grade Java/Spring Boot project. The false positive rate is manageable (especially with LLM confirmation enabled), performance is well within budget, and the config system allows tuning for known limitations. The identified improvements are quality-of-life items, not blockers.
